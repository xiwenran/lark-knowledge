用户正在执行一套完整的知识生产线方案，核心架构如下：

## 系统目标
外部信息输入 → AI 结构化处理 → 表格化收件 → 聚合加工 → 专题知识沉淀 → 内容与产品资产输出

## 三个正式专题
1. 小红书（内容分发、流量获取）
2. 虚拟资料产品（产品化、售卖、变现）
3. AI 编程（提效、自动化、差异化能力）

三者闭环：AI编程提升效率 → 更快生成资料产品 → 通过小红书分发与成交

## 三层分类体系
- 底层逻辑分类：认知/决策/操作
- 专题分类：小红书/虚拟资料产品/AI编程
- 资产形态：方法论/SOP/模板/技巧集/案例包

## 两段式工作流
- 第一段「收件入表」：资料输入 → Claude Code 结构化处理 → 飞书多维表格（24个字段）
- 第二段「聚合升级」：从表格筛选高价值记录 → 聚合 → 知识库草稿 → 回填链接与状态

## 方案完整内容摘要

### 核心执行原则（10条）
1. 不追求全量收集，只追求高价值沉淀
2. 先收件，再升级（不能直接写知识库）
3. 表格是后台数据库，知识库是前台成品区
4. 三层分类必须分层管理，不能混用
5. 知识库只放成品，不放原料
6. 知识库以聚合升级为主，不以单条摘要入库为主
7. 来源不同不影响最终成品结构
8. 权限受限资料不得做原文镜像存档
9. 保存"加工后的知识资产"，不是"来源素材副本"
10. 最终沉淀必须服务于内容生产与产品生产

### 升级条件（必须同时满足）
- 专题归属明确（小红书/虚拟资料产品/AI编程）
- 资产形态明确（方法论/SOP/模板/技巧集/案例包）
- 综合评分 ≥ 4
- 处理状态 = "待升级"
