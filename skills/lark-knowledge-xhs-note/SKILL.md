# lark-knowledge-xhs-note (v1.0.0)

**CRITICAL — 开始前 MUST 先用 Read 工具读取 [`../lark-shared/SKILL.md`](../lark-shared/SKILL.md)**

## 功能定位

根据用户指定的飞书链接或内容，撰写一篇可直接发布的小红书笔记。

**触发方式**：手动触发，用户说「写小红书笔记」或「/小红书笔记」+ 飞书链接 / 粘贴内容。

---

## Step 1: 获取源内容

**飞书链接**：
```bash
lark-cli docs +fetch --doc "<飞书链接>" --format markdown
```

**直接内容**：用户直接粘贴的文字，原样使用。

读完后提炼核心信息：
- **产品/主题是什么**：一句话说清楚
- **关键数据**：价格、销量、时间、数量等具体数字
- **核心结论**：最值得传播的 1-2 个洞察
- **目标读者**：谁会感兴趣

---

## Step 2: 撰写笔记

### 硬性约束

| 项目 | 要求 |
|------|------|
| 标题 | **≤ 20 字**，带数字或结果感 |
| 正文 | **≤ 300 字**，分 3-5 个短段落 |
| 标签 | **5-10 个**小红书话题标签（#xxx） |

### 写作风格（铁律，每条都必须遵守）

**1. 口语化 —— 像跟朋友聊天**
- ✅「我试了一下，真的能跑通」
- ❌「经测试验证，该流程可完整运行」

**2. 具体数字说话 —— 有数字就用数字**
- ✅「9.9 元卖了 4633 份」
- ❌「以较低价格获得了大量销售」

**3. 一句话一个意思 —— 不绕弯**
- 每句话不超过 25 字
- 不用从句套从句

**4. 举例驱动 —— 用场景让人马上能懂**
- ✅「比如你是班主任，期末要写 50 份评语」
- ❌「该产品适用于多种教育场景」

**5. 结果感强 —— 让人一眼知道看完能得到什么**
- 标题要有「动作 + 结果」结构
- ✅「拆了一个年会员PPT产品，4633单的玩法原来这么简单」
- ❌「PPT年会员产品分析报告」

**6. 不要营销腔**
- ❌ 禁止：「强烈推荐」「必看」「绝绝子」「yyds」「家人们」
- ✅ 允许：「我觉得」「说实话」「你们可以试试」

### 内容合规（小红书平台规则）

- 内容定位为**拆解/分析/分享经验**，不是广告推荐
- 用「拆解/分析/观察/分享」等中性词
- 禁止「推荐/种草/安利/必买」等导购措辞
- 不写虚假数据、不夸大效果

### 标题写法公式

从以下公式中选最合适的：
- `数字 + 结果`：「9.9元卖4633单，这PPT资料包真能赚」
- `拆解 + 发现`：「拆了个教师资料号，月入5千的路子很清晰」
- `我 + 动作 + 结果`：「我试了3天虚拟资料变现，第1单来了」
- `疑问 + 答案`：「PPT模板怎么卖？拆完这个号我懂了」

### 正文结构模板

```
【开头】1-2 句，抛出最抓人的数据或结论（≤60 字）

【展开】2-3 段，拆解核心要点，每段一个小论点（≤180 字）
- 可以用 emoji 分隔要点（📌 💡 ✅）
- 每个论点配一个具体数字或例子

【收尾】1 句，总结或引导互动（≤60 字）
- 「你觉得这个赛道还能做吗？」
- 「感兴趣的可以看我主页完整拆解」
```

---

## Step 3: 输出与确认

直接展示完整笔记供用户确认：

```
📌 标题：<≤20字标题>

<正文，≤300字>

#标签1 #标签2 #标签3 #标签4 #标签5 #标签6 #标签7 ...
```

用户确认后可直接复制发布。如需调整，按用户反馈修改后重新输出。
