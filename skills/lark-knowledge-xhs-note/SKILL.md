# lark-knowledge-xhs-note (v1.0.0)

**CRITICAL — 开始前 MUST 先用 Read 工具读取 [`../lark-shared/SKILL.md`](../lark-shared/SKILL.md)**

## 功能定位

根据用户指定的飞书链接或内容，撰写一篇可直接发布的小红书笔记。

**触发方式**：手动触发，用户说「写小红书笔记」或「/小红书笔记」+ 飞书链接 / 粘贴内容。

---

## Step 1: 获取源内容

**飞书链接**：
```bash
lark-cli docs +fetch --doc "<飞书链接>" --format markdown
```

**直接内容**：用户直接粘贴的文字，原样使用。

读完后判断内容类型，提炼核心信息：

### 1a — 判断内容类型

根据页面来源和内容自动判断，走不同的撰写风格：

| 内容类型 | 判断依据 | 笔记风格 |
|---------|---------|---------|
| **商品拆解** | 页面来自「小红书虚拟产品」专题，或含价格/销量/产品形态等商业数据 | 拆解视角：分析商业模式，用「拆解/分析/观察」措辞 |
| **行业洞察** | 页面来自「All In Podcast」知识库，或含投资/市场/科技趋势等内容 | 知识分享视角：传递信息和判断，用「聊聊/分享/看看」措辞 |
| **经验分享** | 页面来自其他专题（小红书/AI编程等），或含方法论/SOP/实操经验 | 经验分享视角：讲自己做了什么、学到了什么 |

**判断不确定时**，问用户「这篇内容按哪个方向写？」

### 1b — 提炼核心信息

- **主题是什么**：一句话说清楚
- **关键数据**：价格、销量、时间、数量等具体数字
- **核心结论**：最值得传播的 1-2 个洞察
- **目标读者**：谁会感兴趣

## Step 2: 撰写笔记

### 硬性约束（所有类型通用）

| 项目 | 要求 |
|------|------|
| 首段 | **≤ 30 字**，无独立标题，首段即标题（品牌/来源 + 主题钩子） |
| 正文 | **≤ 300 字**（含首段），分 3-5 个短段落 |
| 标签 | **5-10 个**小红书话题标签（#xxx） |

> **为什么没有独立标题**：小红书笔记正文第一段自动显示为标题预览。独立标题字段 ≤ 20 字太短，放不下品牌名+期号+主题。改为首段 ≤ 30 字，一句话同时完成品牌标识和内容钩子。

### 通用写作风格（铁律，所有类型都必须遵守）

**1. 口语化 —— 像跟朋友聊天**
- ✅「我试了一下，真的能跑通」
- ❌「经测试验证，该流程可完整运行」

**2. 具体数字说话 —— 有数字就用数字**
- ✅「9.9 元卖了 4633 份」
- ❌「以较低价格获得了大量销售」

**3. 一句话一个意思 —— 不绕弯**
- 每句话不超过 25 字
- 不用从句套从句

**4. 举例驱动 —— 用场景让人马上能懂**
- ✅「比如你是班主任，期末要写 50 份评语」
- ❌「该产品适用于多种教育场景」

**5. 结果感强 —— 让人一眼知道看完能得到什么**
- 首段要有「动作 + 结果」或「现象 + 悬念」结构

**6. 不要营销腔**
- ❌ 禁止：「强烈推荐」「必看」「绝绝子」「yyds」「家人们」
- ✅ 允许：「我觉得」「说实话」「你们可以试试」

### 分类型差异化风格

#### 🔍 商品拆解型

**定位**：以实操经验分享者身份，拆解具体虚拟产品的底层逻辑和方法论。不是旁观者分析，是自己做过的人在分享。

**数据来源说明**：飞书表格中不一定有具体销量数据，不能编造。重点是**底层逻辑方法论+实操经验分享**，而非数据报告。

**内容合规**：
- 用「拆解/分析/观察/研究」等中性词
- **禁止**「推荐/种草/安利/必买」等导购措辞
- 不写虚假数据、不夸大效果

**首段**（≤ 30 字，方法论切入：经验背景+核心发现）：
- 示例：`做了半年教师资料，定价的底层逻辑就1条`
- 示例：`虚拟资料这行，选品比内容重要10倍`
- 示例：`这个虚拟资料包9块9，怎么卖了4000多单`

> 标题公式仅供参考方向，根据内容灵活写，不必须套用。

**正文结构**（围绕一个洞察点，3层递进）：
```
【首段】≤ 30 字，方法论切入
【正文】围绕一个洞察点，3层递进：
  ① 这个判断是什么（一句话说清楚）
    示例：「它卖的不是PPT。买家花的不是9块9买50页PPT，而是买"省掉3小时备课时间"」
  ② 为什么这么说（数据/案例/亲身经历，让人信服）
    示例：「一开始我也觉得资料越多越好，后来发现买家只看前3页」
  ③ 对你意味着什么（翻译成行动结论）
    示例：「最值得学的一个设计：详情页第2张图不放内容预览，放的是一张效率对比图。卖效率，不卖内容」
【结尾】自然收束，不引导评论
  示例：「这条路子跑通了，后面有新进展再更」
  示例：「下篇聊聊流量端怎么搞，比产品本身难多了」
```

**钩子位置**：正文不放引流话术。通过**评论区置顶评论**完成引导（如"完整拆解报告私信'拆解'"）。

**段落过渡要求**：段落之间要有自然衔接，不能像博客一样突然跳到下一个话题。用"但更聪明的是…"、"定价也很聪明…"、"最值得学的…"这种口语化过渡。

#### 💡 行业洞察型（All In Podcast 等）

**定位**：单点深挖，围绕一个洞察点讲透，给读者一个完整的"原来如此"体验。不是期号概览，不是多观点平铺。

**嘉宾知名度说明**：不能假设读者认识 Chamath/Sacks 等人名。标题用"洞察点本身"切入，不依赖人名吸引点击。嘉宾信息放正文内简短介绍。

**内容合规**：
- 用「聊聊/分享/看看/了解一下」等分享类措辞
- 可以带个人判断（「我觉得这个趋势很重要」）

**首段**（≤ 30 字，用洞察点本身切入，不是期号概览）：
- 示例：`SaaS 不是在转型，是在等死——只是它自己不知道`
- 示例：`算力竞赛的钱打到哪去了？答案和你想的不一样`
- 示例：`All In E265 说了一件事，比国内大多数分析早了半年`

> 标题公式仅供参考方向，根据内容灵活写，不必须套用。

**正文结构**（围绕一个洞察点，3层递进）：
```
【首段】≤ 30 字，用洞察点本身切入（不是期号概览）
  示例：「SaaS 不是在转型，是在等死——只是它自己不知道」
  示例：「算力竞赛的钱打到哪去了？答案和你想的不一样」

【正文】围绕一个点，3层递进：
  ① 这个判断是什么（一句话说清楚）
  ② 为什么这么说（数据/案例/类比，让人信服）
  ③ 对你意味着什么（翻译成行动结论）

【结尾】自然收束，不引导评论
  - 「这个判断值不值得信，你自己想想」
  - 「下一篇聊这期另一个让我重听3遍的点」
```

**钩子位置**：正文不放引流话术。通过**评论区置顶评论**引导。正文最后一张配图放 PDF/笔记资料截图预览，视觉暗示"还有完整版"。

**段落过渡要求**：段落之间必须有自然衔接。禁止从一个话题直接跳到另一个话题。用口语化连接，让3层递进读起来像一个完整论证，不像三个独立片段。

#### 📝 经验分享型

**定位**：讲自己做了什么、踩了什么坑、学到了什么。

**内容合规**：
- 用第一人称，讲真实经历
- 可以分享工具和方法，但不是推销

**首段**（≤ 30 字）：
- 示例：`我试了3天虚拟资料变现，第1单来了`
- 示例：`做小红书3个月，最大的坑是这个`
- 示例：`用AI写笔记，效率翻了3倍不夸张`

**正文结构**：
```
【首段】≤ 30 字，概括经历和结果
【正文】2-3 个核心经验/教训，每个配具体例子（≤150 字）
【结尾】自然收束（≤60 字）
  示例：「感兴趣可以看我主页，有完整复盘」
```

### 信息密度3铁律（新增通用规则，所有类型必须遵守）

1. **数字替换形容词**：写完后搜"很/非常/大量/优质"，强制换成具体数字
2. **每句携带1个可转述信息单元**：念给朋友听，每句他能复述出"那一个点"
3. **冰山句式**：`[量化背景] + 冒号 + [直接结论]`，背景压在数字里

### 钩子体系（新增通用规则）

**核心原则**：正文不放引流话术（避免违规），钩子全部通过**评论区置顶评论**完成。

- 商品拆解型评论区置顶：`完整拆解+操作手册 私信「拆解」发你📎`
- 行业洞察型评论区置顶：`完整中英对照笔记+精华整理 私信「All In」发你📎`
- 正文最后一张配图放资料截图预览，视觉暗示"有完整版"

## Step 3: 配图打包

根据内容类型，自动准备配图素材。**所有配图统一输出到子文件夹**：

```
~/Downloads/AllInPodcast/<期号>_<主题关键词>_<序号>/
```

示例（同一期第1篇）：`~/Downloads/AllInPodcast/E270_算力之战_1/`
示例（同一期第2篇）：`~/Downloads/AllInPodcast/E270_算力之战_2/`

> 子文件夹命名规则：期号 + 下划线 + 首段主题关键词（2-4 字）+ 下划线 + 序号，如 `E270_算力之战_1`、`E270_算力之战_2`、`E265_Chamath狠话_1`。同一期的不同洞察角度用 `_1`、`_2`、`_3` 后缀区分。文件夹不存在时自动创建。

### 3.0 — 封面生成（脚本一键生成）

使用 `scripts/cover-generator/generate.py` 脚本，输入标题+期号，输出 1080×1350 深色封面 PNG。

**两套模板**（SVG 模板已定稿，视觉风格固定，只替换文字内容）：

| 模板 | 配色 | 品牌标识 |
|------|------|----------|
| `breakdown` | 深墨绿底 + 翡翠绿强调 | `虚拟产品实操手记` + `BREAKDOWN · NO.XXX` |
| `allin` | 深蓝黑底 + 金色强调 | `ALL IN PODCAST` + `ALL IN · VOL.XXX` |

**用法**：

```bash
# 商品拆解封面
DYLD_LIBRARY_PATH=/opt/homebrew/lib python3 scripts/cover-generator/generate.py breakdown \
  --title "第一行" "第二行" "高亮关键词" \
  --subtitle "副标题第一行" "副标题第二行" \
  --info "产品信息 · 价格 · 销量" \
  --number 012 \
  -o <配图子文件夹>/cover.png

# All In 封面
DYLD_LIBRARY_PATH=/opt/homebrew/lib python3 scripts/cover-generator/generate.py allin \
  --title "第一行" "第二行" "高亮关键词" \
  --subtitle "副标题第一行" "副标题第二行" \
  --info "All In Podcast · E270" \
  --number 270 \
  -o <配图子文件夹>/cover.png
```

**标题规则**：最后一行自动用强调色高亮（breakdown=翡翠绿，allin=金色），把最有冲击力的词放最后一行。

**依赖**：`brew install cairo` + `pip install cairosvg`

**模板文件**：`scripts/cover-generator/allin.svg` / `breakdown.svg`（修改模板视觉风格时编辑这两个 SVG）。

### 3a — 商品拆解型：拆解笔记图

如果 upgrade Step 7 已生成拆解笔记图（`/tmp/sketchnote_<编号>_*.png`），**复制到子文件夹**。
如果还没有，提示用户「要不要先生成拆解笔记图？」

### 3b — 行业洞察型（All In）：手绘笔记 + PDF 预览截图

**手绘笔记**：查找已生成的 sketchnote 图片（`/tmp/allin_<期号>_sketch_*.png`），**复制到子文件夹**。

如果找不到图片，或者文件名是旧结构（`核心议题`/`市场判断`/`四人立场`/`国内启示`），需要用 `generate_sketchnote.py` 重新生成：

```bash
DYLD_LIBRARY_PATH=/opt/homebrew/lib python3 ~/lark-knowledge/scripts/allin/generate_sketchnote.py \
  --record-id "<record_id>" --output-dir /tmp
```

新结构输出 5 张：封面 + 本期最反直觉的判断 + 嘉宾最激烈的分歧 + 说回咱们 + 完整资料预览。

**PDF 内容预览截图**（产品购买钩子）：

从已生成的 PDF 中截取一页作为预览图，吸引用户购买完整版：

```bash
# 先创建子文件夹
mkdir -p ~/Downloads/AllInPodcast/<期号>_<主题关键词>

# 用 Python 从 PDF 截取多页为图片
python3 -c "
import fitz  # PyMuPDF
import os
out_dir = os.path.expanduser('~/Downloads/AllInPodcast/<期号>_<主题关键词>')
doc = fitz.open('<PDF路径>')
for idx, page_num in enumerate([<页码1>, <页码2>, <页码3>]):
    page = doc[page_num]
    pix = page.get_pixmap(matrix=fitz.Matrix(2, 2))
    out = f'{out_dir}/pdf_preview_{idx+1}.png'
    pix.save(out)
    print(f'ok: {out} ({len(pix.tobytes()) // 1024} KB)')
"
```

**截取 3 页**，从以下候选中按优先级选取：
1. **封面页** —— 干净整洁，第一张图就有品质感
2. **信息最密集的页面**（如精华金句页、深度分析页、中英对照页）—— 内容密度高、排版好看，展示专业度；不硬绑定"五维分析概览页"，按实际页面内容选最能展示价值的
3. **某一页中英对照逐字稿** —— 只选 1 页展示排版质感，不会泄露完整内容

**不能选的**：连续多页逐字稿 —— 等于白送核心付费内容。1 页展示排版是钩子，多页就是赠品了。

> 💡 如果 PyMuPDF 未安装，提示用户：`pip3 install PyMuPDF`
