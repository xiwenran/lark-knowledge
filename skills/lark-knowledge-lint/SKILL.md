# lark-knowledge-lint (v1.0.0)

**CRITICAL — 开始前 MUST 先用 Read 工具读取：**
1. `../lark-shared/SKILL.md` — 认证、权限处理
2. `~/.agents/skills/lark-knowledge-config/config.json` — base_token、table_id、字段名

## 功能定位

定期对知识库做「体检」，发现四类问题：

| 检查项 | 说明 |
|--------|------|
| A. 积压预警 | 待升级超过 30 天的记录 |
| B. 孤岛词条 | 已升级但没有相关词条链接的页面 |
| C. 关键词缺口 | 高频关键词没有对应专题页面 |
| D. 矛盾检测 | 同专题记录的 AI摘要存在潜在观点矛盾（可选，深度巡检） |

**触发词**：`/巡检`（默认执行 A+B+C）；`/巡检 深度` 追加 D。

**配置来源**：`~/.agents/skills/lark-knowledge-config/config.json`

---

## Step 1: 拉取全量表格记录

```bash
lark-cli base +record-list \
  --base-token "<config.base.base_token>" \
  --table-id "<config.base.table_id>"
```

提取以下字段备用：
`record_id`、`标题`、`收录时间`、`专题归属`、`资产形态`、`关键词标签`、`处理状态`、`综合评分`、`知识库页面链接`、`AI摘要`

---

## Step 2: 检查 A — 积压预警

筛选：`处理状态 = 待升级` AND `收录时间` < 今天 − 30天。

输出：积压条目的标题、收录时间、综合评分、专题归属。
优先级排序：综合评分高的排前面。

---

## Step 3: 检查 B — 孤岛词条

筛选：`处理状态 = 已升级` AND `知识库页面链接 ≠ 空`。

对**最近 20 条**（按收录时间倒序）逐一检查页面：

```bash
lark-cli docs +fetch --doc "<知识库页面链接>" --format markdown
```

检查页面内容是否包含 `## 相关词条`：
- **有** → 正常
- **没有** → 标记为孤岛词条

> 限制最多 20 条，避免 token 消耗过大。如需全量扫描，用户可追加参数 `/巡检 全量`。

---

## Step 4: 检查 C — 关键词缺口

统计所有记录的 `关键词标签`，找出：
- 出现频次 ≥ 3 次的关键词
- 但在所有已升级记录的 `标题` 中没有出现该关键词的独立词条页面

输出：关键词、出现次数、建议新建页面的专题归属。

---

## Step 5: 检查 D — 矛盾检测（仅深度巡检时执行）

按 `专题归属 + 资产形态` 分组，对每组内的 `AI摘要` 进行 LLM 对比分析：

- 找出观点直接矛盾的条目（如：同一工具的正反评价、同一方法的不同结论）
- **不 fetch 页面内容**，仅用表格中的 AI摘要 字段，避免大量 token 消耗
- 每组只处理 AI摘要 字符数最多的 10 条

输出：矛盾对（记录A vs 记录B）、矛盾点简述、建议处理方式（合并 / 注明版本差异 / 保留争议）。

---

## Step 6: 输出巡检报告

```
知识库巡检报告  <YYYY-MM-DD>
━━━━━━━━━━━━━━━━━━━━━━━━━━━━

A. 积压预警（<N> 条）
  ⚠️ <标题>  收录于 <日期>  评分 <分>  专题：<专题>
  ...
  （若无则显示：✅ 无积压）

B. 孤岛词条（已检查 <N> 条，发现 <M> 个孤岛）
  🔗 <标题>  <链接>
  ...
  （若无则显示：✅ 已检查页面均有相关词条）

C. 关键词缺口（建议新建 <N> 个专题页面）
  📌 "<关键词>"  出现 <N> 次  建议专题：<专题>
  ...
  （若无则显示：✅ 无明显缺口）

D. 矛盾检测（仅深度巡检时）
  ⚡ <标题A> vs <标题B>  矛盾点：<简述>  建议：<处理方式>
  ...

━━━━━━━━━━━━━━━━━━━━━━━━━━━━
建议下一步：
1. 优先升级积压中评分最高的 <标题>
2. 为孤岛词条补充相关词条（可触发 /升级 重新处理或手动补充）
3. 考虑新建 "<关键词>" 专题页面
```
