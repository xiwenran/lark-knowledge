# All In Podcast 收件入表

**CRITICAL — 开始前 MUST 先用 Read 工具读取：**
1. `~/.agents/skills/lark-knowledge-config/config.json` → 读 `all_in_podcast` 区块，获取 base_token、table_id、字段 ID
2. `../lark-shared/SKILL.md` — 认证、权限处理

## 适用场景

用户提供 All In Podcast 某期 YouTube 链接，一键完成：
- 抓取字幕/描述
- 四段式 AI 流水线生成五维分析摘要
- 写入飞书节目收件表

---

## 工作流

```
YouTube URL → 抓取内容 → Worker（结构提取）→ Haiku（关键段校对）→ Writer（五维分析）→ 冷眼（审查）→ 用户确认 → 写表
```

---

## Step 1: 抓取字幕与元数据

### 1A. Jina Reader（首选）

```
WebFetch: https://r.jina.ai/https://www.youtube.com/watch?v=<视频ID>
```

Jina 返回的内容通常包含：视频标题、发布日期、描述、自动字幕（如有）。

**判断是否拿到字幕**：返回内容超过 2000 字且有明显对话片段 = 字幕成功；若只有描述段落（< 500 字）= 未获取字幕，走 1B。

### 1B. yt-dlp 下载字幕（Jina 未获取字幕时）

```bash
# 下载自动生成字幕（英文）
yt-dlp --write-auto-sub --sub-lang en --sub-format vtt --skip-download \
  --output "/tmp/allin_%(id)s" "<YouTube URL>"
```

字幕文件路径：`/tmp/allin_<视频ID>.en.vtt`

读取 vtt 文件后，用正则去掉时间码行，合并成连续文本：
```
grep -v "^[0-9]" /tmp/allin_<id>.en.vtt | grep -v "WEBVTT" | grep -v "^$" | grep -v "-->" | tr '\n' ' '
```

### 1C. 同步抓取 YouTube 元数据

无论走 1A 还是 1B，同时用 WebFetch 抓取视频页面获取：

| 字段 | 提取位置 |
|------|---------|
| 英文原标题 | `<title>` 标签或 og:title |
| 发布日期 | `datePublished` 或页面显示日期 |
| YouTube 播放量 | 页面显示播放量（转为纯数字，不含逗号） |
| 时长 | `contentUrl` 的 duration 或页面显示时长（转换为分钟数，取整） |
| 期号 | 从标题提取：`E<数字>` 或 `#<数字>` 格式；若无则留空 |

## Step 2: Worker（结构提取 + 五维草稿）

**执行身份**：Sonnet 4.6 子代理

**任务**：基于 Step 1 获取的字幕文本，完成两件事：

### 2A. 确认元数据
核实从页面抓到的标题、日期、播放量、时长，补充期号（若未提取到）。

### 2B. 生成五维分析草稿

All In Podcast 专用五维框架（与通用知识库5维框架不同）：

| 维度 | 核心问题 | 写法指引 |
|------|---------|---------|
| **① 议题背景** | 本期讨论了什么，背景是什么？ | 1-2 句，说清时代背景与触发事件 |
| **② 论点链** | 主要论点是什么，推理链条如何？ | 3-5 个核心论点，按逻辑顺序排列，每条 1 句 |
| **③ 市场判断** | 涉及哪些市场趋势、机会或风险？ | 具体到行业/公司/数据，避免泛泛 |
| **④ 四人立场** | Chamath / Jason / Sacks / Friedberg 各自核心观点？ | 每人 1-2 句，有分歧时着重体现，无明显分歧可合并描述 |
| **⑤ 国内启示** | 对中国读者/投资者/创业者的参考价值是什么？ | 结合国内语境做类比或对照，不照翻，要迁移 |

**草稿字数**：300-500 字，五维各一段，用 `①②③④⑤` 编号开头。

### 2C. 判断主题分类

从以下选项中选一个最匹配：
- `科技&AI`：AI、半导体、软件、产品、工程
- `全球视野`：地缘政治、国际局势、宏观经济（非金融市场）
- `金融&市场`：股市、加密货币、风险投资、利率、宏观金融
- `商业&创业`：商业模式、并购、消费、创业案例
- `人物专访`：有明确嘉宾的专访期

**输出草稿**供 Step 3 校对使用。

---

## Step 3: Haiku 校对（关键段抽样检查）

**执行身份**：Haiku 4.5 子代理

**任务**：不做全文翻译检查，只做**关键段抽样校对**：

### 3A. 强制检查项（逐一核对）

对五维分析草稿中出现的所有以下内容，回到字幕原文逐一核实：
- **具体数字**（市值、增速、金额、比例）
- **公司名 / 人名**（拼写、归属是否正确）
- **强判断句**（"XX 会崩"、"XX 是 20 年来最大机会"等）

发现不符的地方标注 `[❓需核实: 原文是「...」]`，不擅自修改。

### 3B. 随机抽样（10% 字幕段落）

从字幕全文随机抽取约 10% 的段落，验证 Worker 的概括是否与原文一致。发现重大偏差则标注。

### 3C. 输出校对报告

```
校对报告：
- 强制检查：X 处，问题 Y 个（列出）
- 随机抽样：Z 段，偏差 W 个（列出）
- 整体可信度：高 / 中 / 低
```

将报告与修订意见传给 Step 4。

---

## Step 4: Writer（五维分析定稿）

**执行身份**：Sonnet 4.6 子代理

**任务**：基于 Worker 草稿 + Haiku 校对报告，完成终稿：

1. 修正 Haiku 标出的所有 `❓` 问题
2. 对 Haiku 标注"整体可信度：低"的内容做保守化处理（降低确定性语气）
3. 将五维分析精炼为**150-250 字**的最终版本，保持 `①②③④⑤` 结构
4. 同步生成：
   - **精华金句**：从字幕中摘取 3-5 句最有代表性的原话（英文），附 1 句中文译文
   - **五维综合评分**（1-10）：基于议题重要性 + 论点深度 + 国内启示价值综合打分

**评分参考**：
- 9-10：重大议题，论点深刻，国内高度可迁移
- 7-8：重要议题，有较强洞察，部分可迁移
- 5-6：一般议题，有信息量，迁移价值有限
- 1-4：纯娱乐/吐槽，实质内容少

---

## Step 5: 冷眼审查

**执行身份**：独立 Sonnet 4.6 子代理（不看前三步的推理过程，只看最终产出）

**任务**：只读 Step 4 产出的终稿，独立判断：

1. 五维覆盖是否完整（有无维度偷懒或空话）
2. ④四人立场是否有明显张冠李戴
3. ⑤国内启示是否真的是"迁移"而不是硬套
4. 精华金句是否真的是金句，还是普通句

给出：`通过 ✅` 或 `需修改 ⚠️（具体指出）`。

若需修改，Writer 修订后**不再走第二轮冷眼**，直接进 Step 6。

---

## Step 6: 用户确认 + 写入收件表

### 6A. 展示摘要供用户确认

```
─────────────────────────────────────
📻 All In Podcast 收件预览
─────────────────────────────────────
期号：E xxx
标题：[英文原标题]
中文标题：[建议中文标题]
发布：YYYY-MM-DD | 时长：xx 分钟 | 播放量：xx 万
主题分类：xxx | 五维评分：x/10
冷眼审查：通过 ✅ / 需修改 ⚠️

五维分析：
① 议题背景：...
② 论点链：...
③ 市场判断：...
④ 四人立场：...
⑤ 国内启示：...

精华金句：
"..." — [说话人]（译：...）
─────────────────────────────────────
确认写入请回复「写入」，需修改请指出。
```

### 6B. 敏感内容检查（写入前）

对照三级标准，决定「发布状态」默认值：

| 级别 | 判断标准 | 默认发布状态 |
|------|---------|------------|
| 🟢 安全 | 科技/AI/商业/创业为主 | `未发布`（可直接推进） |
| 🟡 注意 | 涉及宏观经济/VC监管/中美贸易 | `未发布` + 在「审核备注」注明"需调整角度" |
| 🔴 暂缓 | 政治/军事/中美冲突为主轴 | `未发布` + 在「审核备注」注明"暂不上小红书" |

### 6C. 写入节目收件表

```bash
lark-cli base +record-upsert \
  --base-token "<config.all_in_podcast.base_token>" \
  --table-id "<config.all_in_podcast.table_id>" \
  --json '{
    "期号": "E xxx",
    "英文原标题": "...",
    "中文标题": "...",
    "发布日期": "YYYY-MM-DD",
    "YouTube链接": "https://www.youtube.com/watch?v=xxx",
    "时长（分钟）": <数字>,
    "YouTube播放量": <数字>,
    "主题分类": "科技&AI",
    "翻译状态": "待处理",
    "注释状态": "待处理",
    "Haiku审核": "通过",
    "PDF状态": "未生成",
    "发布状态": "未发布",
    "五维综合评分": <数字>,
    "审核备注": "..."
  }'
```

> 注意：`飞书页面URL` 此时留空，待 P3-C 逐字稿生成后回填。

### 6D. 完成汇总

```
收件完成 ✅
E xxx | [标题] | 主题：[分类] | 五维评分：x/10
record_id：<id>
```

## 注意事项

- **字幕长度**：All In 单期通常 60-120 分钟，字幕约 15000-30000 词。Step 2 Worker 处理时可将字幕分段后并行分析，最后综合。
- **无字幕情况**：若 yt-dlp 也没有字幕（极少数私有/会员期），只用描述文本生成，在五维分析开头注明「基于视频描述，未含逐字稿」。
- **播放量格式**：统一写入纯数字（如 `3800000`），不含万/千分位。
- **中文标题**：由 Writer 基于英文标题和五维分析提炼，简洁直观，不超过 20 字。
